Closing A1C Care Gaps Through Intelligent EHR Integration

A backend FHIR application that reads A1C observation data from eClinicalWorks to identify open, met, and excluded diabetes care gaps — supporting quality measure reporting and population health workflows.

📊

Real-Time Gap Status

Reads A1C Observations via FHIR R4 to classify each patient's care gap as Open, Met, or Excluded in real time.

🔐

SMART Backend Services

Uses OAuth 2.0 Client Credentials with RS384 JWT assertions. No PHI is stored outside the authorized eCW environment.

🏥

eClinicalWorks Native

Designed specifically for eCW FHIR R4 endpoints. Compatible with production and sandbox environments.

📋

Quality Measure Ready

Outputs gap status aligned with HEDIS and CMS quality measures for HbA1c (Diabetes Care measure set).

Security & Compliance

This application accesses protected health information (PHI) under a HIPAA Business Associate Agreement with eClinicalWorks. All data in transit is encrypted via TLS 1.2+. No PHI is persisted outside the authorized eCW FHIR environment.
